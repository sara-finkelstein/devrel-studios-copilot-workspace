## Friction Points
<!-- Things that take multiple attempts, require manual steps, or slow you down -->

- **OneNote can't be safely automated on this machine** — Work IQ has no OneNote endpoints at all, and desktop COM automation (the same trick that fixed live Excel edits) fails with a "Library not registered" error even after manually loading the typelib. Root cause looks like a broken Click-to-Run typelib registration, not fixable without an Office repair. Compounded by a real prior incident: a past auto-write overwrote and overlapped Sara's existing OneNote content (sync conflict). *Takeaway:* never write to OneNote programmatically for now — stage content in a plain file/digest and let Sara paste it in herself. Revisit COM only if an Office repair/reinstall happens.

- **Skill PRs from stale repo state** — packaging skills without first syncing `mvp-copilot-plugins` upstream can create outdated counts/overlaps and lead to avoidable review rejection. *Takeaway:* fetch/rebase first, then branch and submit. Recurred 7/15: local checkout's `origin` remote was Sara's own fork, frozen ~2 months behind real `microsoft/mvp-copilot-plugins` (`upstream` remote) — she was rebasing/showing work against the stale fork without realizing fork ≠ upstream. Root cause of Golnaz's "wrong repo version" comment. *Suggested skill:* "make sure I'm up to date" — fetch upstream, sync fork, rebase local branch, surface only real content conflicts for a decision (never resolve shared-file conflicts unilaterally). Promote at 3+ uses (now at 2).
- **YouTube transcript mode mistaken for subtitle track output** — after uploading a corrected caption file, transcript view looked different and appeared "wrong" even though the standard caption track was uploaded successfully. *Takeaway:* always verify from the player CC track picker first, not transcript panel rendering.
- **YouTube OAuth app is in test mode → ~7-day token expiry** — refresh tokens die about weekly ("Token has been expired or revoked"), forcing a browser re-auth (`node ~/.copilot/youtube-tokens/auth.js <channel>`, listens on localhost:3847). **Real fix:** publish the Google Cloud OAuth app (team-lead/admin task) so tokens stop expiring. *Takeaway:* if an Analytics/Data API call suddenly 401s, re-auth first before debugging.
- **Doc generate → close-in-Word → regenerate loop** — generating the FY26 report from a Python script meant every tweak required Sara to close the doc in Word or hit `PermissionError`/`-v2` fallback saves, and she couldn't hand-edit without being clobbered on regen. **Resolved:** switched to doc-as-source-of-truth ("Path A") — Sara hand-edits the OneDrive master, Copilot stops regenerating and hands over paste-ready snippets instead. *Takeaway:* for collaborative/near-final docs, hand editing beats regeneration; offer the switch early. (Recurred 7/9 on the MVP Unplugged doc — resolved by saving to a `(v2)` filename.)
- **Accepting YouTube collaboration invites** — repeatedly a sticking point across sessions: confusion over which account/channel must be active and which direction the accept URL goes. Resolved (collaborator-first URL + collaborator as active channel); now documented in the youtube-collaborator skill and learnings.md.
- **Having to close Excel every time Copilot edits a spreadsheet** — openpyxl can't write to a file that's open in Excel (`PermissionError`), so Sara had to repeatedly close/reopen "Infra Series - Producer Booking Sheet.xlsx" across a single session. **Resolved 7/20:** edit through a live COM connection instead (`win32com.client.GetActiveObject('Excel.Application')`, find the workbook by name in `excel.Workbooks`, edit `Range`/`Cells` directly, then `wb.Save()`) — the file never has to close, and Sara can watch edits happen live. Now the default method for editing open Excel files for this user (see learnings.md).

## Skill Candidates
<!-- Patterns ready to be turned into reusable skills -->

- **Run-of-show vs. final ADO Episode title reconciliation** — Sara wants to compare a preloaded run-of-show doc (draft/placeholder titles) against final Episode titles that get added live during a show, to catch a disconnect — exact nature of the mismatch still TBD pending a conversation with coworkers (7/15). *Next step:* revisit once Sara knows what the disconnect actually is (wrong titles vs. missing episodes vs. mapping errors) before building anything.

- **Daily topic-based inbox filing + digest** — sort inbox emails matching a topic (e.g. "Microsoft Fabric") into a named mail folder (e.g. Fabric TTF), then send an end-of-day summary of the actual discussion content. Set up as a recurring scheduled prompt (weekdays 5pm) on 7/13 via `manage_schedule`: search `/me/messages` with `$search`, verify each hit's subject/bodyPreview to filter false positives, move genuine matches via `/me/messages/{id}/move`, stay quiet if nothing new. Seen 1x (7/13, Fabric TTF folder). *Suggested skill:* generalize to "file inbox emails about [topic] into [folder] and digest daily" for any topic/folder pair. Promote at 3+ uses.

- **Caption QA + upload verification checklist** — recurring friction in "is this the right track?" after subtitle uploads (transcript view vs CC track picker). *Suggested skill:* run post-upload checks (track kind, last updated, expected terms), then provide a quick human verification checklist in YouTube player. Promote at 3+ uses.
- **YouTube Studio CTR export walkthrough** — recurring need to export the Advanced Mode Content CSV (Impressions + Impressions click-through rate %) to feed the Content Autopsy. Studio video-ID column = `Content` (trim leading space on IDs starting with `-`), skip the `Total` row, and **filter the table** to dodge the ~500-row export cap. Seen 2x (7/8, 7/9). *Suggested skill:* a guided "export CTR for [these videos]" checklist. Promote at 3+ uses.

- **Capture meeting follow-ups into the to-do list** — pull action items from a recorded/transcribed meeting (via Work IQ), then drop them into the project-grouped `To-Do-List.xlsx` under the right project band. Seen 1x (6/24, MVP intro chat w/ Martin Tatar). *Suggested skill:* "capture my action items from [meeting] into my to-do list" — find meeting via Work IQ, extract owner/task/due, map to existing project section (or create one), set Status=Open. Promote at 3+ uses.

- **Add series candidate presenter** — research a person (org chart position/manager chain, location, public talks or recent work) → confirm findings with Sara → write a formatted row into the relevant booking spreadsheet's candidate tab → mirror the context into the project's `projects.md` entry. Seen 3x in one session (7/17, Infra Series: Sajay Antony, Akash Singhal, Karthik Uthaman org check). Sara's take: "not sure there's a lot to be gained by adding this all together, but good to keep in mind" — low conviction it's worth automating yet. *Suggested skill:* "add [name] as a series candidate" — single command chains the lookup + spreadsheet write + context update. Hold off promoting; revisit only if it recurs across other series/shows, not just Infra Series.
